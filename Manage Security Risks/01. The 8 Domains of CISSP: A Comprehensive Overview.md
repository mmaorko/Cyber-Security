# Domain 4: Communications and network security: 
This domain centers on securing physical networks and wireless communications to prevent unauthorized exposure. It involves analyzing user behavior—such as stopping employees 
from using unsecured hotspots—and creating strict network policies to keep data transmissions safe.
Securing data in motion and Includes network protocols, firewalls, VPNs, segmentation, wireless security, and attacks like MITM or sniffing.

### Network Security: Protecting the Infrastructure
Network security deals with how systems are connected and how traffic flows between them.
#### It includes:
- Network segmentation
- Firewalls and access control lists
- Intrusion detection and prevention systems
- Secure routing and switching
- Network monitoring

A well-designed network limits trust. Critical systems are separated from public-facing services. Internal traffic is controlled, not assumed safe. If one segment is compromised, the rest of the environment should remain protected.

### Communication Security: Protecting the Data in Transit
Communication security ensures that data traveling across networks cannot be read or modified by unauthorized parties.
#### Common mechanisms include:
- Encryption
- Secure communication protocols
- Virtual Private Networks (VPNs)
- Secure email transmission
For example, when you see HTTPS in a browser, it relies on Transport Layer Security (TLS) to encrypt the connection between the client and the server. This prevents interception and tampering during transmission.

### Short Summary:
Communication and Network Security focuses on protecting data as it moves across networks. In today’s environment, nearly every business process depends on digital communication—web applications, cloud services, email, remote access, online payments. The moment data leaves a device and travels across a network, it becomes exposed to risk.
This domain ensures that data in transit remains confidential, intact, and accessible only to authorized parties.

# Domain 5: Identity and Access Management: 
This domain focuses on ensuring that only the right people have access to specific data and systems. It covers the technical and administrative processes used to identify, authenticate, and authorize users. For a security analyst, this involves managing user permissions and implementing tools like Multi-Factor Authentication (MFA) to prevent unauthorized entry.
#### The Core Concept
IAM answers three fundamental questions:
- Who are you? – Verifying the user or system’s identity (Authentication)
- What can you do? – Determining allowed actions (Authorization)
- What data can you access? – Restricting access to resources (Access Control)
Without proper controls, unauthorized users can gain access to sensitive data or critical systems.
### Short summary: 
Who can access a system, what they can do, and when they can do it. It's Covers authentication, authorization, accounting (AAA), SSO, identity lifecycle, and access control models
(RBAC, DAC, MAC).

# Domain 6: Security Assessment and testing : 
The Security Assessment and Testing domain is all about being proactive. Instead of waiting for a breach to happen, this domain focuses on identifying and fixing risks, threats, and vulnerabilities before they can be exploited.

Think of it as a "stress test" for an organization's digital defenses. Security assessments help leadership understand whether their internal systems are truly solid or if they are sitting on hidden risks. To do this, organizations often bring in penetration testers (or "pen testers"). These are ethical hackers whose job is to find the cracks in the armor that a real threat actor might use to get inside.

### The Core Concept

#### Security Assessment and Testing answers three key questions:
- Is the system secure? – Evaluate the current security posture
- Are the controls working? – Verify that each security control is effective
- Where are the weaknesses? – Identify vulnerabilities and potential threats

### Assessment vs Testing
#### Security Assessment: 
- A structured evaluation of the environment.
- Reviews policies, procedures, architecture, and controls.
- Includes risk analysis and compliance checks.
#### Security Testing: 
- A technical, hands-on approach.
- Simulates attacks to discover vulnerabilities.
- Measures what would happen if a real attack occurred.

### Short summary: 
Security Assessment and Testing is the process of evaluating a system, application, or network to determine how secure it really is. In simple terms, it answers the question: “How safe is this system, and where are the weaknesses?”
This domain ensures that security controls are functioning properly and that vulnerabilities or risks are identified before attackers can exploit them.
It Verifying that controls actually work.Includes vulnerability scanning, penetration testing, audits, log reviews, and security metrics.

### Security Operations Center (SOC)

Security Operations (SecOps) is the domain where an organization’s day-to-day security activities are executed. Simply put, it’s the central hub for monitoring, 
managing, and responding to security threats and incidents on a continuous basis.
While Risk Management or IAM defines what needs protection and who should have access, Security Operations ensures that these plans are actively enforced, 
vulnerabilities are monitored, and incidents are handled promptly.

#### SOC responsibilities include:
- Real-time monitoring of networks and systems
- Integrating threat intelligence feeds
- Event correlation and anomaly detection
- Prioritizing and coordinating incident response
- Maintaining logs and audit trails

### includes using strategies, processes, and tools such as:
- Training and awareness: Educating employees to recognize threats and follow security protocols.
- Reporting and documentation: Keeping clear records of incidents and notifying the necessary stakeholders.
- Intrusion detection and prevention: Monitoring networks for unauthorized access and blocking malicious activity.
- SIEM tools (Security Information and Event Management): Software that aggregates and analyzes log data to identify security threats in real-time.
- Log management: The process of collecting, storing, and reviewing records of system events.
- Incident management: The structured approach used to respond to and manage the aftermath of a security breach.
- Playbooks: Pre-defined, step-by-step guides on how to respond to specific types of security incidents.
- Post-breach forensics: The scientific collection and analysis of digital evidence to determine how a breach happened.
- Reflecting on lessons learned: Reviewing an incident after it is resolved to improve future defenses and processes.

The cybersecurity professionals involved in this domain work as a team to manage, prevent, and investigate threats, risks, and vulnerabilities. 
These individuals are trained to handle active attacks, such as large amounts of data being accessed from an organization's internal network 
outside of normal working hours. Once a threat is identified, the team works diligently to keep private data and information safe from threat actors.

# Domain 8: Software development and security : 
Software Development Security focuses on ensuring that security is integrated into every phase of software creation and deployment. History has shown that many major 
security breaches originate from unsafe code or design flaws. This domain addresses those risks by embedding security throughout the development lifecycle.

It is not just about writing code; it’s about making security an integral part of the entire development process.

### Core Concept
#### Software Development Security answers three critical questions:
- Is the software designed securely? – Detect threats and vulnerabilities during the design phase
- Is the code implemented securely? – Follow safe coding practices during development
- Is the deployed software protected? – Maintain security during deployment and runtime

### Short summary: 
Building secure applications and its Covers SDLC, secure coding, code reviews, application vulnerabilities (OWASP), and DevSecOps concepts.
