# Asset:
Any item that holds value to an organization. This can include data, systems, hardware, software, intellectual property, or even personnel.

# Attack Vectors:
The methods or pathways attackers use to gain unauthorized access to systems, networks, or data.

# Authentication:
The process of verifying an individual’s identity before granting access to a system.

# Authorization:
The process of granting an authenticated user permission to access specific resources or perform certain actions.

# Availability:
The principle that data and systems must be accessible to authorized users when needed.

# Biometrics:
Unique physical characteristics, such as fingerprints or facial recognition, used to verify identity.

# Confidentiality:
The principle that only authorized individuals should have access to specific data or assets.

# Confidentiality, Integrity, Availability (CIA) Triad:
A foundational security model that guides organizations in managing risk and designing systems and policies to protect information.

# Detect:
A NIST core function focused on identifying potential security incidents and improving monitoring capabilities to increase detection speed and effectiveness.

# Encryption:
The process of converting readable data into an encoded format to prevent unauthorized access or tampering.

# Govern:
A NIST core function that ensures cybersecurity strategies, policies, roles, and risk management processes align with organizational goals and regulatory requirements.

# Identify:
A NIST core function centered on understanding cybersecurity risks and their impact on organizational assets and operations.

# Integrity:
The principle that data must remain accurate, authentic, and reliable throughout its lifecycle.

# National Institute of Standards and Technology (NIST) Cybersecurity Framework (CSF):
A voluntary framework consisting of standards, guidelines, and best practices designed to help organizations manage cybersecurity risk.

# National Institute of Standards and Technology (NIST) Special Publication (SP) 800-53:
A comprehensive set of security and privacy controls developed to protect federal information systems in the United States.

# Open Web Application Security Project (OWASP):
A nonprofit organization dedicated to improving software security through open-source tools, resources, and community collaboration.

# Protect:
A NIST core function focused on implementing safeguards—such as policies, training, and technical controls—to reduce cybersecurity risk.

# Recover:
A NIST core function concerned with restoring systems and operations to normal after a cybersecurity incident.

# Respond:
A NIST core function that ensures appropriate actions are taken to contain, analyze, and remediate security incidents.

# Risk:
Anything that has the potential to negatively impact the confidentiality, integrity, or availability of an asset.

# Security Audit:
A structured review of an organization’s security controls, policies, and procedures against defined standards or expectations.

# Security Controls:
Safeguards or countermeasures designed to reduce specific security risks.

# Security Frameworks:
Structured guidelines that help organizations design and implement security strategies to mitigate risk and protect data.

# Security Posture:
An organization’s overall ability to protect critical assets and respond effectively to security challenges.

# Threat:
Any event, condition, or actor that has the potential to cause harm to organizational assets.
