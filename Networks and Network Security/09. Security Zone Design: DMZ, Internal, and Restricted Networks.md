# Security Zone
A Security Zone is a segment of a network that protects the internal network from the internet. It is part of a technique called Network Segmentation — which divides the network into 
separate segments, each with its own access permissions and security rules.

## Security Zones category
categorizing areas as Controlled or Uncontrolled is the most basic way to define where your security authority begins and ends.

## Uncontrolled Zone
The Uncontrolled Zone is any network that exists outside your organization’s. It refers to the entire world outside your organization’s control boundary.
In simple terms, it is a network where you have no administrative authority, and from which attacks can originate at any time. Biggest example is The public internet


### Why is it a Huge Risk?
Any data packet coming from this zone may be malicious. No IP address or identity can be fully trusted (risk of IP spoofing) and Your ability to monitor traffic in this zone is limited

### Ingress Filtering:
Ensure that no private IP addresses (e.g., 192.168.x.x) from external zones enter your network. This is a basic defense against IP spoofing.
- **_Geoblocking:_** If your company operates only in Bangladesh, blocking IP ranges from other countries (e.g., Russia or North Korea) is a common practice.
- **_DDoS Protection:_** If there is a sudden surge of abnormal traffic from the Uncontrolled Zone, it may indicate a Denial-of-Service (DoS/DDoS) attack.

### The Real Problem 
Many organizations expose their servers directly to the Uncontrolled Zone (public internet).
This is like sleeping with your front door open.

**_As a security analyst, your job is to:_**
Move those servers into a DMZ and Reduce direct public exposure as much as possible



## Controlled Zone
A Controlled Zone is a subnet that protects the internal network from the outside world (Uncontrolled Zone/Internet). Security tools such as firewalls, IPS (Intrusion Prevention Systems), and others are deployed here to filter traffic or data flow.

### Layers of the Controlled Zone (The Architecture)

**_In a professional network, the Controlled Zone is divided into three main parts:_**

### 1. DMZ (Demilitarized Zone)

This is the outermost layer of the Controlled Zone. Servers that need to communicate with the internet are placed here.
- **_What it contains:_** Web servers, DNS, email gateway, proxy servers
- **_Purpose:_** Acts as a “buffer” or protective wall for the internal network

### 2. Internal Network
This is the company’s actual private area, where employee PCs and local servers reside.
- **_Purpose:_** It does not accept direct inbound connections from the internet and Access is only possible through the DMZ or via VPN.

### 3. Restricted Zone
This is the deepest and most secure part of the Controlled Zone.
- **_What it contains:_** Databases (Customer DB), financial records, and highly confidential data
- **_Purpose:_** Only privileged users can access this zone and It is protected by an additional dedicated firewall layer.

### The Real Problem

Main challenges regarding the Controlled Zone will be:
- **_Strict ACLs (Access Control Lists):_** Rules between zones must be very strict. For example, a web server in the DMZ should not directly access a database in the Restricted Zone.
There must be an intermediate application layer.
- **_Micro-segmentation:_** In modern security, zoning alone is not enough. Each server within a zone is further isolated (micro-segmentation) so that if one server is compromised, the attacker cannot move laterally to others.
- **_Logging & Monitoring:_** All cross-zone traffic must be logged. If you observe someone attempting direct file transfer (SFTP) from the internal network to the Restricted Zone, it becomes a major investigation point.

### Real Risk
If your Controlled Zone design is weak (for example, no firewall between the DMZ and Internal Network), an attacker can compromise a simple web server and gain control over all company PCs and servers. This is called a “Flat Network,” which is a nightmare for security.
