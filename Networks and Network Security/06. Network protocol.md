What is a Network Protocol?
Networks need rules to function properly. These rules are called network protocols — a set of guidelines that two or more devices follow to determine the order of delivery and 
structure of data sent across a network.
Imagine you type www.yourcomany.org into your browser. Just to load that one website, your device uses four different protocols working together behind the scenes.

### Domain Name System(DNS): 
It is a Communication protocols that translates internet domain names into IP addresses, because Computers do not understand human-readable names; they only understand IP addresses. When you type a domain name, DNS translates it into an IP address (e.g., 192.0.2.1).

#### **_Security View:_**
- Attack Process:
An attacker injects false information into a DNS server. For example, the real IP of mybank.com (1.1.1.1) is replaced with a fake IP (2.2.2.2) controlled by the attacker.

- Result:
When users type the bank’s website, they are redirected to a fake website created by the attacker and may unknowingly enter their login credentials.

- What Analysts Check:
If a domain suddenly resolves to an unfamiliar IP address or a geographically unusual location, it is a red flag.

### Transmission Control Protocol (TCP): 
TCP is a Communication protocols. Before any data is exchanged, your device and the web server must "meet" each other. TCP handles this by performing a handshake — a verification process that confirms both 
devices are ready to communicate. Only after this handshake does data start flowing.
It uses a 3-Way Handshake (SYN → SYN-ACK → ACK) to ensure both sides are ready for data communication.

#### **_Security View:_**
- Attack Process:
The attacker sends thousands of SYN requests to a server. The server responds with SYN-ACK and waits for the final ACK (creating half-open connections).

- Result:
The attacker never sends the final ACK. The server’s resources get exhausted maintaining these incomplete connections, preventing legitimate users from accessing the service.

- What Analysts Check:
If a SIEM tool shows a high number of incomplete TCP handshakes from a single source IP, that IP should be quickly blocked.

### Address Resolution Protocol (ARP): 
Data travels across the network in packets, hopping through multiple routers. ARP figures out the MAC address of the next router or device along the path, making sure the data 
reaches the right destination.

#### **_Security View:_**
The biggest weakness of the ARP protocol is that it trusts any reply without verification.
- Attack Process:
An attacker inside a local network tells the router, “I am the victim user,” and tells the victim, “I am the router.”
- Result:
All network traffic is routed through the attacker’s machine. This is known as an on-path (Man-in-the-Middle) attack. The attacker can steal passwords or modify data.
- What Analysts Check:
If a network scan shows the same MAC address assigned to multiple IP addresses, it is a strong indication of ARP poisoning.

### HyperText Transfer Protocol Secure (HTTPS)
Once the connection is established, HTTPS handles the actual request for the webpage. It provides a secure, encrypted channel between your browser and the web server
using SSL/TLS technology, keeping your data safe from attackers. and it is a Security Protocols.

#### **_Security View:_**

Attackers now use HTTPS to hide malicious traffic. Since the traffic is encrypted, traditional firewalls or intrusion prevention systems (IPS) often cannot see what is inside the data, allowing malware to pass through undetected.

 **_How Is This Tackled?_**
Large organizations use a technique called SSL/TLS Inspection (Break and Inspect). Encrypted traffic is intercepted at the gatewayIt is decrypted and inspected for threats
Then it is re-encrypted and sent to its destination. This allows security systems to detect hidden malware within encrypted traffic.

**_Common Misconception_**

Many people believe that if a website uses HTTPS, it is completely safe.

Reality:
HTTPS only ensures that the connection is encrypted. It does not guarantee that the website itself is trustworthy.
Attackers can easily create phishing sites (e.g., secure-login-bank.com) and obtain free SSL certificates from services like Let’s Encrypt, enabling HTTPS on malicious websites.

## Three categories of network protocols
Network protocols can be divided into three main categories: communication protocols, management protocols, and security protocols. There are dozens of different network protocols, but you don’t need to memorize all of them for an entry-level security analyst role. However, it’s important for you to know the ones listed in this reading. 


## Communication protocols
### User Datagram Protocol (UDP)
UDP is a “fire and forget” protocol. It does not perform any handshake before sending data, which makes it extremely fast but less reliable.

#### Why is it used?
It is ideal for applications like video streaming, online gaming, and DNS requests. In these cases, losing a packet does not break the entire system, and speed is more important than perfect accuracy.

#### Security Risk (UDP Amplification Attack):
Since UDP does not verify sessions, attackers can spoof the source IP address and send small requests that generate large responses. This technique is often used in DDoS attacks.

#### Layer:
UDP operates at the Transport Layer of the TCP/IP model.

### Hypertext Transfer Protocol (HTTP)
HTTP is the core protocol used for browsing websites. However, it is a clear text protocol, meaning all data transmitted through it can be read by anyone intercepting the traffic.

#### Why is it insecure?
When you log in to a website using HTTP, your username and password are transmitted as plain text over the network. Attackers can easily capture this data using packet sniffing techniques.

#### HTTPS (The Secure Version):
Most modern websites use HTTPS, which adds SSL/TLS encryption to secure the data. It typically uses port 443.

#### Layer:
HTTP operates at the Application Layer of the TCP/IP model.

## Network Management Protocols
Network management protocols act as the “health monitors” of your infrastructure. As a security professional, understanding them is critical because attackers often target these protocols to gather information or gain control over network devices.

### Simple Network Management Protocol (SNMP)
SNMP is a powerful protocol used to remotely monitor and manage devices such as routers, switches, and servers.

#### Functions:
It can monitor bandwidth usage, reset passwords, and even modify device configurations.

#### Layer:
SNMP operates at the Application Layer of the TCP/IP model.

#### Security Risks (The Real Problem)
- SNMP v1 and v2c:
These versions use community strings (like passwords) transmitted in plain text.
If an attacker captures this string, they can gain full control over your network devices.
- SNMP v3:
Supports encryption and strong authentication, making it significantly more secure.
Security professionals strongly recommend using SNMPv3.

### Internet Control Message Protocol (ICMP)
ICMP is mainly used for network troubleshooting and error reporting. It does not carry user data but instead provides information about network conditions.

#### Functions:
The most common use is the ping command, which checks whether a device is online and measures latency.

#### Layer:
ICMP operates at the Internet Layer of the TCP/IP model.

#### Security Risks
- Reconnaissance:
Attackers use ping or traceroute to map your network and identify active devices.
- ICMP Flood (DDoS):
Sending a large number of ICMP requests can overwhelm a server and cause denial of service.
- ICMP Tunneling:
Attackers may hide data inside ICMP packets to bypass firewalls and exfiltrate information.

As Security Specialist, keep these in mind:

- Disable Unnecessary ICMP:
Block external ICMP (ping) requests where possible to reduce network visibility to attackers.
- SNMP Hardening:
Avoid default community strings like “public” or “private”. These are common and easily exploited.
- Monitor Logs:
If your SIEM shows repeated SNMP requests from unknown IPs, it may indicate an attempt to access device configurations.
Real Risk (Business Impact)

If SNMP is left unsecured, an attacker can modify network device configurations (e.g., switches) and redirect all traffic to their own system (Man-in-the-Middle attack). This can severely compromise the organization’s data security and lead to catastrophic consequences.

## Security Protocols
Security protocols are network protocols that ensure that data is sent and received securely across a network. Security protocols use encryption algorithms to protect data in transit. Below are some common security protocols.

### SFTP (Secure File Transfer Protocol)
SFTP is a protocol used for secure file transfer and is built on top of SSH (Secure Shell).

#### Key Difference:
In regular FTP, usernames and passwords are sent in plain text, which makes it highly insecure.
In contrast, SFTP encrypts the entire session, making it much safer for transferring sensitive data.

#### Encryption:
SFTP uses strong encryption methods such as AES (Advanced Encryption Standard) to protect data.

#### Port:
It typically uses TCP Port 22, which is the default port for SSH.

#### Cloud Connection:
As mentioned in your notes, SFTP is commonly used in cloud storage systems. Whenever you upload or download files from the cloud, SFTP is often used to ensure secure data transfer.
