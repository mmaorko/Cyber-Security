# Subnetting & CIDR 
Subnetting is the process of dividing one large network into smaller, organized groups called subnets. For example if your company's network is a large city, each subnet is a 
distinct neighborhood within that city. Each subnet is defined by a unique combination of IP address + network mask, effectively creating a "network within a network."

## Why Subnetting?
- **_Efficiency:_** When devices on the same subnet communicate, the switch keeps transmissions within that subnet, improving speed.
- **_Security_** Creates isolated zones (like the DMZ, Internal, and Restricted zones you learned earlier).
- **_Cost Saving_** No need to request additional network IP addresses from your ISP.

## What is CIDR?
Classless Inter-Domain Routing (CIDR) is a method of assigning subnet masks to IP addresses to create subnets. It replaced the old classful system and expanded the number of 
available IPv4 addresses.

### Format:
```
Regular IPv4:   198.51.100.0
CIDR IPv4:      198.51.100.0/24
                              ↑
                     IP Network Prefix
```

The number after the slash (/24) is called the IP network prefix — it tells you how large the subnet is.
```
198.51.100.0/24
= covers all IPs from 198.51.100.0
               to   198.51.100.255
= 256 total addresses in this subnet
```

## Quick Summary

| Term                    | Description                                                     |
| ----------------------- | --------------------------------------------------------------- |
| **Subnetting**          | Dividing a large network into smaller subnets                   |
| **Subnet**              | A smaller network within a larger network                       |
| **Network Mask**        | Defines the boundary of a subnet                                |
| **CIDR**                | A modern and flexible method for subnet addressing              |
| **IP Network Prefix**   | The `/number` at the end of a CIDR address (e.g., /24)          |
| **Classful Addressing** | An old 1980s system that divided IP addresses into Class A to E |
