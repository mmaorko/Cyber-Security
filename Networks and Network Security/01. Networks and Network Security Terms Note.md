# Basic Notes
## Baseline configuration (baseline image):
A secure, documented system setup that serves as a standard reference for future builds, releases, and updates.

## Hardware:
The physical components of a computer system, whose security is essential for overall system protection.

## Multi-factor authentication (MFA):
An authentication method that requires users to verify their identity using two or more factors, significantly reducing the risk of unauthorized access.

## Network log analysis:
The process of analyzing network logs to detect suspicious activities and identify potential security incidents.

## Operating system (OS):
The interface between users and hardware, where proper configuration and maintenance are critical for system security.

## Patch update:
An update applied to software or operating systems to fix known vulnerabilities and improve security.

## Penetration testing (pen test):
A controlled and authorized simulated attack used to identify vulnerabilities before they can be exploited.

## Security hardening:
The practice of reducing a system’s attack surface by eliminating unnecessary services, ports, and access points.

## Security information and event management (SIEM):
A centralized solution that collects and analyzes log data to enable real-time monitoring and threat detection.

## World-writable file:
A file that can be modified by any user, posing a significant security risk.

## Active Packet Sniffing:
A type of attack where data packets are intercepted and modified during transit to compromise network communications.

## Address Resolution Protocol (ARP):
Address Resolution Protocol (ARP) is a very important networking protocol that works as a bridge between the Data Link Layer (Layer 2) and the Network Layer (Layer 3) of the OSI model. In simple terms, it is used to find the corresponding MAC address (physical address) from a given IP address.
How ARP Works (The Mechanism)
When a device (Host A) wants to communicate with another device (Host B) on the same local network, it may know Host B’s IP address but not its MAC address. The following steps occur:

#### ARP Request:
Host A sends a broadcast message to the entire network:
- Who has the IP address 10.0.0.5? Please send me your MAC address.
#### ARP Reply:
The device with IP address 10.0.0.5 (Host B) responds directly:
- This is my IP, and my MAC address is AA:BB:CC:11:22:33.
#### ARP Cache:
- Host A stores this information in its ARP table (cache) so it doesn’t need to send repeated requests in the future.

## Bandwidth:
The maximum data transmission capacity of a network, measured in bits per second (bps).

## Baseline Configuration:
A documented set of system specifications that serves as a reference for future builds, updates, or deployments.

## Bluetooth:
A short-range wireless protocol enabling communication between nearby devices.

## Botnet:
A network of compromised computers controlled by a single threat actor, commonly called a “bot herder.”

## Cloud-Based Firewalls:
Firewalls hosted by cloud service providers that filter and monitor traffic for cloud-based resources.

## Cloud Computing:
The use of remote servers and services via the internet instead of relying on local hardware.

## Cloud Network:
A network of servers in remote data centers accessible via the internet to store and process data.

## Controlled Zone:
A subnet or network segment that enforces security controls to protect internal resources from external threats.

## Data Packet:
The fundamental unit of data transmitted across a network.

## Denial of Service (DoS) Attack:
A Denial-of-Service (DoS) attack is a cyber attack in which an attacker tries to make a computer, server, or network resource unavailable to legitimate users. It primarily involves crashing the system or making it extremely slow.
The main difference from a DDoS attack is that in a DoS attack, the attacker usually launches the attack from a single computer or source.

## Distributed Denial of Service (DDoS) Attack:
Distributed Denial of Service attack is an advanced and much more powerful form of a Denial of Service (DoS) attack. Instead of using a single computer, a hacker uses thousands or even millions of malware-infected devices on the internet (called a botnet) to flood a specific target or server with traffic.
The main goal is to exhaust the server’s bandwidth or resources (CPU/RAM) so that normal users can no longer access the website or service.

## Domain Name System (DNS):
A protocol that translates human-readable domain names into IP addresses.

## Encapsulation:
A method used by VPNs and network protocols to protect data by wrapping it inside additional packets for secure transmission.

## File Transfer Protocol (FTP):
A standard protocol used to transfer files between devices over a network.

## Firewall:
A Firewall is a network security system that monitors and controls incoming and outgoing network traffic based on predefined security rules.
In simple terms, it acts as a “digital gate” or “security wall” between your private network and the outside world (the internet).

## Forward Proxy Server:
A Forward Proxy Server is an intermediary server that sits between users in an internal network and the internet. In simple terms, when you browse the internet, your request does not go directly to the website. Instead, it first goes to the proxy server, which then communicates with the website on your behalf.
It mainly operates on the client side (user side).

## Hardware:
Physical components of a computer system.

## Hub:
A network device that broadcasts traffic to all connected devices within a local network.

## Hypertext Transfer Protocol (HTTP):
An application-layer protocol enabling communication between clients and web servers.

## Hypertext Transfer Protocol Secure (HTTPS):
A secure variant of HTTP that encrypts traffic using TLS/SSL for data protection.

## Identity and Access Management (IAM):
A framework of policies, technologies, and processes that ensures the right individuals access the right resources securely.

## IEEE 802.11 (Wi-Fi):
Standards defining wireless local area network (WLAN) communication.

## Internet Control Message Protocol (ICMP):
ICMP is a very important networking protocol mainly used for error reporting and diagnostics in a network. It operates at the Network Layer (Layer 3) of the OSI model.
In simple terms, when the IP (Internet Protocol) sends data packets, ICMP is responsible for informing whether the packet successfully reached its destination or if any problem occurred along the way.

## ICMP Flood:
ICMP Flood (or Ping Flood) is a type of Denial of Service (DoS) attack in which an attacker sends a massive number of ICMP Echo Request packets (commonly known as “ping”) to a target server or device.
The main goal of this attack is to exhaust the target system’s bandwidth and resources (CPU/Memory) so that legitimate users can no longer access the service.

## Internet Protocol (IP):
A set of standards for routing and addressing packets across networks.

## IP Address:
A unique identifier assigned to a networked device.

## IP Spoofing:
IP Spoofing is a type of cyber attack in which an attacker modifies the source IP address of a data packet sent over the internet. As a result, it appears that the packet is coming from a trusted or legitimate source, even though it is actually sent by a hacker.
In simple terms, it is like stealing a digital identity to gain access to a network.

## Local Area Network (LAN):
A network covering a small physical area, such as an office or home.

## Media Access Control (MAC) Address:
A unique identifier assigned to a device’s network interface card.

## Modem:
A device that connects a local network to the internet.

## Multi-Factor Authentication (MFA):
A security method requiring multiple forms of identity verification to access a system.

## Network:
A collection of connected devices that share data and resources.

## Network Log Analysis:
Network log analysis is the process of examining records (logs) of all traffic and events that pass through a network. As a security analyst, your main task is to identify any unusual patterns or security threats from this large volume of data.
In simple terms, network logs are the “digital footprint” of your network, which helps you understand who did what, when, and from where.

## Network Protocols:
Rules defining how data is transmitted and interpreted between devices.

## Network Segmentation:
Network Segmentation is the practice of dividing a large network into smaller sub-networks or “segments.” It is one of the most powerful cybersecurity strategies, primarily aimed at preventing an attack from spreading across the entire network.
In simple terms, it is like dividing a large open room into smaller separate cabins, where each cabin has its own locked door.

## Operating System (OS):
Software that manages hardware resources and provides an interface for users and applications.

## Open Systems Interconnection (OSI) Model:
A conceptual framework describing the seven layers of network communication.

## On-Path Attack:
An On-path attack (formerly known as a Man-in-the-Middle (MITM) attack) is a type of cyber attack where a hacker secretly positions themselves between two communicating parties (for example, your computer and a website).
The attacker acts as a digital “middleman,” allowing them to intercept, monitor, and even modify the data being exchanged between the two parties.

## Packet Sniffing:
Packet Sniffing is the process of capturing and inspecting data packets that travel through a network. While it is a legitimate activity for network troubleshooting or security auditing, hackers can use it to steal passwords, credit card numbers, or other sensitive information.
In simple terms, packet sniffing is like digital eavesdropping—listening in on someone else’s conversation without their knowledge.

## Passive Packet Sniffing:
Observing network traffic without altering it.

## Patch Update:
In simple terms, if a bug or security vulnerability is found in any software, developers release a small piece of code or an update to quickly fix it. This is called a patch.
As a cybersecurity professional, patch management is one of the most important parts of your daily work.

## Penetration Testing (Pen Test):
Penetration Testing (or Pen Testing for short) is an authorized and planned simulated cyber attack used to test the security of a system, network, or application.
In simple terms, just as a hacker would try to break into your system, a pen tester performs attacks in the same way to identify vulnerabilities in the system.

## Ping of Death:
Ping of Death (PoD) is a type of classic Denial of Service (DoS) attack, in which an attacker sends an IP packet to a target computer that exceeds the maximum size allowed by the IP protocol. This attack primarily targets the Network Layer (Layer 3).
As a security professional, you should know that although modern operating systems (OS) can now defend against this attack, it remains a very important concept in the history of network security.

## Port:
A logical endpoint for organizing network traffic between devices.

## Port Filtering:
Port filtering is a security mechanism that allows or blocks incoming and outgoing network traffic based on their port numbers. It is primarily a core function of a firewall.
In simple terms, it is like your network’s “digital gateway”, where a guard checks which door (port) a packet is trying to enter through. If the port is not allowed, the packet is dropped.

## Proxy Server:
A Proxy Server is a system that acts as an intermediary or “gateway” between your computer and the internet. When you visit a website, your request does not go directly to the website; instead, it first goes to the proxy server. The proxy server then retrieves the information from the website on your behalf and delivers it to you.

## Replay Attack:
A Replay Attack is a type of network attack where a hacker secretly intercepts valid communication or data transfer between two parties and later resends the same data. The goal is to trick the system into performing an unauthorized action or granting access.

## Reverse Proxy Server:
A Reverse Proxy Server is an intermediary server that sits between users on the internet and your internal web server. While a forward proxy represents the user, a reverse proxy represents the web server.
In simple terms, the outside world thinks they are accessing your website directly, but they are actually communicating with the reverse proxy. The proxy then receives their requests, fetches the data from an internal server, and delivers it to the user.

## Router:
A network device that directs traffic between multiple networks.

## Secure File Transfer Protocol (SFTP):
SFTP is a secure protocol used for transferring files over a network. It is built on top of the SSH (Secure Shell) protocol, which encrypts the entire communication session.
As a security professional, you should know that traditional FTP, which sends data in plain text, is highly risky in today’s world. That’s why SFTP is the standard for file transfers at the enterprise level.

## Secure Shell (SSH):
SSH is a cryptographic network protocol that ensures secure and encrypted communication between two devices over an insecure network (such as the internet). As a security professional, when you manage a Linux server or a network device (router/switch) remotely, SSH is your primary tool.
It is essentially the modern and secure alternative to the older, insecure protocols like Telnet or rlogin.

## Security Hardening:
The process of strengthening systems to reduce vulnerabilities and minimize attack surfaces.

## Security Information and Event Management (SIEM):
Software that collects and analyzes logs to detect threats and monitor critical activity.

## Security Zone:
A Security Zone is a specific part of a network where the same security policies and trust levels are applied. Through network segmentation, we divide the entire network into smaller zones to prevent an attack from spreading across the whole system.

Simple Network Management Protocol (SNMP):
SNMP is an application layer protocol used to monitor and manage various network devices such as routers, switches, servers, printers, and UPS systems.
As a security professional, it is like a double-edged sword—while it helps you keep an eye on the overall health of your network, if misconfigured, it can become a goldmine of information for hackers.

## Smurf Attack:
Smurf Attack is a type of Distributed Denial-of-Service (DDoS) attack, where an attacker uses fake or spoofed IP addresses to send a large number of ICMP (Ping) packets to many devices on a network. As a result, the victim or target server becomes overwhelmed with response packets, causing it to crash or become unavailable to legitimate users.
This attack primarily exploits the network’s amplification capability, turning a small attack into a much larger one.

## Speed:
The rate at which data is transmitted and received, measured in bits per second.

## Stateful Firewall:
A Stateful Firewall (or Stateful Inspection Firewall) is an advanced version of a traditional packet-filtering firewall. It does not just inspect each packet individually; instead, it keeps track of the “state” of a network connection from start to finish.
In simple terms, it is an intelligent security gate. It knows which users are allowed to send traffic out from inside the network and automatically permits the corresponding return traffic from outside.

## Stateless Firewall:
A Stateless Firewall (or Packet Filtering Firewall) is the oldest and most basic type of firewall. It treats each network packet as a completely separate and independent entity. It does not keep track of previous packets or connection history—hence the name “Stateless.”

## Subnetting:
Subnetting is the process of dividing a large network (for example, a Class C network) into smaller logical networks. As a cybersecurity professional, knowing subnetting is essential for network segmentation and creating security zones.
In simple terms, just as a large building is divided into smaller apartments for easier management, subnetting works in the same way for networks.

## Switch:
A network device that forwards traffic to specific devices within a network.

## SYN Flood Attack:
A SYN Flood Attack is a type of Denial-of-Service (DoS) attack that exploits a weakness in the TCP connection “Three-way Handshake” process. In this attack, the attacker sends a large number of SYN (synchronize) requests to the target server but never completes the final step of the handshake.
In simple terms, it’s like calling someone on the phone, and as soon as they pick up, you hang up without saying anything. If you do this thousands of times, all the lines get busy, and legitimate calls cannot get through.

## TCP/IP Model:
The TCP/IP Model (Transmission Control Protocol/Internet Protocol) is the foundation of the modern internet. It is a layered architecture that defines how data travels from one computer to another. While the OSI model is somewhat theoretical, the TCP/IP model is practical and serves as a real-world standard

## Transmission Control Protocol (TCP):
TCP is one of the main protocols of the modern internet, operating at the Transport Layer of the TCP/IP model. It is a connection-oriented protocol, which means it establishes a reliable connection or “handshake” between two devices before data transfer begins.

## TCP 3-Way Handshake:
A three-step process (SYN → SYN-ACK → ACK) to establish a reliable TCP connection.

## Uncontrolled Zone:
An Uncontrolled Zone (or Untrusted Zone) is the part of a network security architecture that is outside the control of your organization or network administrators.
In simple terms, it is the internet.

## User Datagram Protocol (UDP):
A connectionless protocol that transmits data without ensuring delivery.

## Virtual Private Network (VPN):
A Virtual Private Network (VPN) is a security technology that creates a secure and encrypted “tunnel” over an insecure public network (such as the internet). It hides your device’s real IP address and encodes data so that no one can read it while it’s in transit.

## Wide Area Network (WAN):
A network covering large geographic areas, such as cities, regions, or countries.

## Wi-Fi Protected Access (WPA):
A security protocol for wireless networks ensuring encrypted communication.

# MAC vs IP: 
A switch operates using MAC addresses (Data Link Layer), while a router operates using IP addresses (Network Layer).
