# Packet Sniffing
Packet sniffing is the practice of using software tools to observe data as it moves across a network.
Every packet contains:
- Header — sender's and receiver's IP addresses
- Body — valuable information like names, date of birth, personal messages, financial information, and credit card numbers

Security analysts use it legitimately to debug network issues. However, attackers use it to gain unauthorized access and steal information.

## Two Types of Packet Sniffing Attacks
### 1. Passive Packet Sniffing
- Data packets are only read, not modified
- In modern switched networks, only broadcast or same network segment traffic is visible. Attackers often combine this with techniques like ARP poisoning to view more traffic.
- Example: A postal worker maliciously reading someone's mail while delivering it

### 2. Active Packet Sniffing
- Data packets are manipulated in transit
- Attacker positions themselves as a trusted intermediary (gateway or switch-level attacker)
- Packets are redirected to unintended ports or systems
- Packet contents may be altered (e.g., changing a bank account number)
- Can be used to inject malicious data or hijack sessions

---

## Protection Against Packet Sniffing
### Use a VPN (Virtual Private Network)
A VPN creates an encrypted tunnel for all your data traffic. Even if a hacker successfully performs packet sniffing and intercepts your data, they will be unable to read it. 
The information will appear as unreadable gibberish or ciphertext because it is protected by strong encryption. A VPN does not provide complete anonymity. It shifts trust from the ISP to the VPN provider, and logging policies must be considered.

### Ensure HTTPS (SSL/TLS Encryption)
Always verify that a website's URL begins with https:// instead of http://. The "S" stands for secure and indicates that the site uses an SSL/TLS certificate. This technology establishes
an encrypted connection between your browser and the web server, ensuring that sensitive data (like passwords or credit card numbers) cannot be captured in plain text. HTTPS ensures encryption but not absolute trust. Attackers can still use techniques like SSL stripping or fake domains. Always verify certificates and domain names.

### Avoid Public Wi-Fi
Free Wi-Fi networks at airports, hotels, or coffee shops are often unencrypted and highly insecure. This environment allows anyone on the same network to easily sniff your traffic. 
If you must use public Wi-Fi, always ensure your VPN is active before browsing to provide an extra layer of security.

## Tools Used in Packet Analysis & Attacks
- Wireshark — packet capture and analysis
- tcpdump — command-line packet analysis
- Ettercap — MITM attack simulation
- Bettercap — advanced network attack and These tools are used for both defensive analysis and offensive security testing in controlled environments.

## The Real Problem:
The biggest danger is often already inside the building. An attacker doesn't need to sit outside on the Wi-Fi if they can compromise a single workstation or plug in a rogue device into an Ethernet port.

**_The Mechanic:_** Once inside, hackers use ARP Poisoning (Address Resolution Protocol). They trick devices on the network (victim and router) into associating the attacker’s MAC address with the gateway IP.
- Attackers send forged ARP replies (Gratuitous ARP)
- Victim’s ARP cache gets poisoned
- Network traffic is redirected through attacker system

## Tips 
When practicing packet sniffing (e.g., using tools like Wireshark), always check whether the captured packets are in clear text or encrypted form.
- If data appears as readable text → it is not encrypted
- If data appears unreadable (ciphertext) → encryption is in place

## Detection Techniques:
- Monitor abnormal ARP traffic (frequent ARP replies)
- Detect duplicate IP/MAC address conflicts
- Use IDS/IPS tools (e.g., Snort, Suricata)
- Analyze unusual network latency or traffic rerouting

# IP Spoofing
IP spoofing is the act of modifying the source IP address of a data packet. A hacker uses an IP address that appears “trusted” or familiar to your network. It’s similar to entering a secure building using someone else’s ID card.

# Major Spoofing Attacks
## 01. On-path Attack 
The attacker positions themselves between two legitimate devices (e.g., client and server).
An On-path Attack (formerly known as a Man-in-the-Middle or MITM attack) is a type of cyberattack where a malicious actor inserts themselves into a conversation between two parties—typically a user and a server—to secretly intercept or alter the communication. Attackers may also downgrade secure connections (SSL stripping) if protections are weak.
The primary danger is that both legitimate parties believe they are communicating directly with each other, unaware that an intruder is controlling the flow of information.
Even encrypted traffic can be intercepted, but without proper certificate validation, attackers may still manipulate sessions.

### How It Works
The attack generally consists of two main phases:

#### 1. Interception
The attacker must force the data to travel through their device instead of the direct path. Common methods include:
- ARP Spoofing: In a local network, the attacker sends fake messages to your router to trick it into thinking the attacker's computer is actually you.
- Rogue Wi-Fi Hotspots: An attacker sets up a fake Wi-Fi network (e.g., "Free Airport Wi-Fi"). When you connect, all your data passes through the attacker's laptop.

#### 2. Decryption and Manipulation
Once the traffic is flowing through the attacker, they can:
- Steal Credentials: If you log into a site that isn't properly secured, the attacker can see your username and password in plain text.
- Inject Malicious Data: The attacker can modify the data being sent. For example, they could redirect you to a fake banking website that looks identical to the real one.

### How to Protect
Because this attack happens "in the middle" of the network, defense focuses on making the data unreadable even if it is intercepted:
- Look for HTTPS: Always ensure the websites you visit use https://. The "S" stands for Secure and ensures that the connection is encrypted. Even if a hacker intercepts the data, it will look like gibberish to them.
- Use a VPN: A Virtual Private Network creates a secure "tunnel" for your data, making it nearly impossible for an on-path attacker to see what you are doing.
- Avoid Public Wi-Fi for Sensitive Tasks: Never check your bank account or enter credit card details while connected to an unsecured public Wi-Fi network.
- Multi-Factor Authentication (MFA): Even if an attacker steals your password via an on-path attack, they still won't be able to log in without your second factor (like a code sent to your phone).

### 02. Replay Attack 
A Replay Attack is a form of network attack that is particularly sneaky because it doesn't necessarily involve "cracking" a password or breaking encryption. Instead, the attacker simply "records" a valid transmission and "plays it back" later.

Think of it like a thief recording the sound of you pressing the buttons on your garage door keypad and then playing that recording back to the receiver later that night to open the door.

### How a Replay Attack Works
- Capture: An authorized user sends a legitimate request to a server
- Interception: The attacker intercepts this data packet. Even if the packet is encrypted and the attacker can't read its contents, they know what the packet does based on the context.
- The Replay: At a later time, the attacker sends that exact same packet to the server again.
- Execution: The server sees a perfectly valid, correctly encrypted packet and assumes it is a new, legitimate request from the original user.

### Real-World Examples
- Remote Keyless Entry: An attacker could capture the signal your key fob sends to unlock your car. Later, when you are gone, they replay that signal to unlock the doors.
- Banking Transactions: If a user sends a request to "Transfer $100 to Account A," an attacker could capture that packet and send it ten more times, potentially draining the user's account even if the attacker can't read the encrypted contents of the packet.

### Prevention Methods
- Session Tokens with Time-Stamps: If a packet arrives that is 5 minutes old, the server will reject it as expired.
- One-Time Passwords (OTP): Since the code changes every time, replaying an old code is useless.
- Nonces (Number used once): The server sends a random, unique string (a "nonce") to the client. The client must include this nonce in their response. Once used, the server discards that nonce, so a replayed packet containing the old nonce will be ignored.
- Sequence Numbers: In protocols like IPsec, each packet is given a serial number. If the server receives a packet with a number it has already seen, it knows it’s a replay and drops it.
- Replay attacks can succeed even against encrypted traffic if mechanisms like nonces or timestamps are not implemented.

## 03. Smurf Attack
A Smurf Attack is a classic and highly effective type of Distributed Denial of Service (DDoS) attack. It is particularly clever because it turns an entire network of innocent devices into an "army" that unintentionally attacks a single target. Modern networks are typically protected against Smurf attacks as routers disable directed broadcasts by default. However, misconfigured networks may still be vulnerable.

It gets its name from the idea of many small "Smurfs" (network devices) banding together to overwhelm a much larger giant (the target server).

### How a Smurf Attack Works
The attack relies on a combination of IP Spoofing and ICMP (Internet Control Message Protocol), commonly known as a Ping.

### 1. The Setup (IP Spoofing)
The attacker identifies the IP address of their victim (the target). They then craft a "Ping" packet, but instead of using their own IP address as the sender, they spoof the sender's address to be the victim's IP.

### 2. The Broadcast
The attacker sends this spoofed Ping packet to an IP Broadcast Address of a large network. A broadcast address is a special address that automatically forwards the message to every single device connected to that network.

### 3. The Multiplication (The Flood)
Every device on that network receives the Ping request. Following standard network protocol, they all respond with a "Ping Reply."
Because the sender's address was spoofed, all these devices send their replies to the victim's IP instead of the attacker.

**_If the attacker sends 1 packet to a network with 500 computers, the victim is suddenly hit with 400 responses at once. By doing this repeatedly, the attacker can completely overwhelm the victim's bandwidth, causing their server to crash._**

## Defend Against Smurf Attacks
- Disable IP Broadcasting: Most modern routers are configured to ignore "directed broadcast" requests from the outside internet. This stops the attacker's single packet from ever reaching the "army" of devices.
- Filter ICMP Traffic: Network administrators can limit or block ICMP (Ping) traffic at the firewall level so the server isn't forced to process thousands of replies.
- Ingress Filtering: As we discussed earlier, firewalls can check if the incoming traffic's source IP makes sense. If a packet claims to be from inside the network but is coming from the outside, it is dropped.
