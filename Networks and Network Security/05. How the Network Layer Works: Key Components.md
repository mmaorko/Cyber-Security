# Operations at the network layer
Network layer operations and the IPv4 packet structure mainly act as the logical guidelines for data transmission. The primary responsibility of the network layer is to ensure end-to-end delivery. 
It mainly performs two major functions:
- **_Addressing:_** Each packet is assigned a source and destination IP address. Without this, routers would not know where to send the packet.
- **_Routing:_** Routers use their routing tables to decide the best path to forward the packet so that it reaches its destination quickly and securely.

## 1. Management & Identification
These fields ensure the packet is recognized and handled correctly if it needs to be broken down.
-	**_Version (4 bits):_** Almost always set to 4 for IPv4.
-	**_IHL (4 bits):_** Tells the receiver where the header ends. Since the "Options" field varies, this is crucial.
One small detail that often trips people up: The IHL (Header Length) doesn't count bytes directly. It counts 32-bit words.
	1. If the header is the standard 20 bytes, the IHL value is actually 5 ($5 \times 4 \text{ bytes} = 20 \text{ bytes}$).
	2. If the IHL is greater than 5, the router knows to look at the Options field.
## 2. Priority & Control
- **_Type of Service (ToS):_** Now often referred to as DSCP (Differentiated Services Code Point). It’s how a router knows to let your "Lag-free" gaming or VoIP call jump to the front of the line ahead of a background file download.

## The Fragmentation Trio
When a large packet hits a network with a smaller MTU (Maximum Transmission Unit)—like moving from an Ethernet cable to a slower WAN link—it must be broken down. These three fields manage that process:
- **_Identification (16 bits)_**
 Think of this as a Serial Number.
1.	Every fragment created from the original large packet carries the exact same Identification number.
2.	Purpose: When the destination computer receives thousands of fragments from different sources, it uses this ID to group all the "pieces" that belong to the same original "picture."

- **_Flags (3 bits)_**
This acts as a Status Indicator for the router and the receiver. It consists of three specific bits:
  1. Reserved Bit: Always set to 0.
  2. Don't Fragment (DF): If this is set to 1, the router is forbidden from breaking the packet. If it's too big to pass, the router drops it and sends an error message.
  3. More Fragments (MF): This is the "To Be Continued" sign.
     - If MF = 1, the receiver knows more pieces are coming.
     - If MF = 0, the receiver knows this is the very last piece of the puzzle.
- **_Fragment Offset_**
This is the Position Marker.
  1. It tells the receiving device exactly where this specific piece fits into the original data stream.
  2. Why it’s needed: Fragments don't always arrive in the correct order (e.g., Piece #3 might arrive before Piece #1). The Offset allows the destination to reassemble them correctly regardless of when they show up.

## 2. Priority & Control
-	**_Type of Service (ToS):_** Now often referred to as DSCP (Differentiated Services Code Point). It’s how a router knows to let your "Lag-free" gaming or VoIP call jump to the front of the line ahead of a background file download.
-	**_Time to Live (TTL):_** The "self-destruct" timer. Without this, a packet stuck in a routing loop would circle the globe forever, eventually clogging the entire internet.
## 3. Addressing & Integrity
-	**_Protocol:_** Identifies what’s inside the "envelope" (usually TCP or UDP).
-	**_Header Checksum:_** A quick math check. If the header was garbled during transit, the router drops it immediately rather than trying to send it to a non-existent address.
-	**_Source/Destination IP:_** The 32-bit logical addresses (e.g., 192.168.1.1).
-	**_Total Length:_** Shows the full size of the IP packet (header + data), up to 65,535 bytes.
-	**_Options:_** This is optional and is used for security testing or source routing.

## Difference between IPv4 and IPv6
IPv4 and IPv6 are the two main addressing systems of the internet. Every device connected to the internet needs a unique IP address. For a long time, IPv4 fulfilled this need, but due to the massive expansion of the internet, its limited number of addresses began to run out (IPv4 Address Exhaustion), which led to the creation of IPv6.
#### **_IPv4 vs IPv6: Key Differences_**

| Feature          | IPv4 (Internet Protocol v4)          | IPv6 (Internet Protocol v6)                        |
| ---------------- | ------------------------------------ | -------------------------------------------------- |
| Address Size     | 32-bit                               | 128-bit                                            |
| Address Format   | Dotted decimal (e.g., 172.16.254.1)  | Hexadecimal (e.g., 2001:0db8:85a3::8a2e:0370:7334) |
| Total Addresses  | Approximately 4.3 billion ($2^{32}$) | 340 undecillion ($2^{128}$)                        |
| Header Structure | 20 to 60 bytes (variable)            | Fixed 40 bytes                                     |
| Security (IPSec) | Optional                             | Built-in or default support                        |
| Configuration    | DHCP or manual                       | Supports auto-configuration (SLAAC)                |


### Why is there a shift to IPv6?

Simply put, IPv4 has a limited number of addresses. 4.3 billion addresses are not enough for today’s billions of smartphones, laptops, and IoT (Internet of Things) devices. On the other hand, IPv6 has such a vast number of addresses that even if every grain of sand on Earth had its own IP, it still wouldn’t run out. This has paved the way for the future expansion of the internet.



### In SOC and network security, these two protocols bring different types of challenges:

**_NAT (Network Address Translation):_** In IPv4, we used to hide many local IPs behind a single public IP, which provided a form of “obscurity” or security. In IPv6, each device can have a global public IP, increasing the risk of direct attacks if the firewall is not properly configured.
**_Simplified Header:_** The IPv6 header is much more organized and does not include a “Checksum” field (as it is handled at a lower layer). This helps routers process packets faster, improving network performance.
**_Dual-Stack Vulnerability:_** Most modern networks support both IPv4 and IPv6 simultaneously. Hackers often exploit this to bypass detection. For example, your firewall might block IPv4 traffic, but if it fails to filter IPv6 traffic, attackers can enter through that path.
The Real Issue

Although the vastness of IPv6 has solved the IP shortage problem, it has introduced new types of security challenges. Hackers often try to bypass security systems using IPv6 transition tunnels. As an analyst, your job is to monitor IPv6 traffic in your SIEM dashboard and ensure that it is properly filtered by the firewall.
