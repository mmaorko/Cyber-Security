# Firewall 
A firewall is a network security device that monitors incoming and outgoing traffic. It either allows or blocks traffic based on a defined set of security rules. One common 
method it uses is port filtering — blocking or allowing specific port numbers. For example:

- Port 443 → allowed for HTTPS
- Port 25 → allowed for Email
- Everything else → blocked

 ## Firewall Types (Deployment Types)
Firewalls can be categorized into three types based on how and where they are deployed: 

### Hardware Firewall:
This is a physical device that is usually placed between the router and the internet. It inspects each packet before it enters the network. It is ideal for large 
organizations because it does not put additional load on individual devices.

### Software Firewall:
This is installed directly on a computer or server. It analyzes the incoming and outgoing traffic of that specific device. It is cost-effective, but it uses the 
computer’s processing power (RAM/CPU).

### Cloud-based Firewall (FaaS):
Cloud providers offer this as a service. It filters traffic in the cloud before it reaches the on-site network. It is essential for protecting cloud assets (such as 
AWS S3 or Azure VM).

## Stateless vs. Stateful Firewall

### Stateless Firewall
A stateless firewall treats each data packet as an independent event. It does not keep track of any previous or subsequent information about the packet.

- **_Operation:_**
It primarily uses Access Control Lists (ACLs). Whenever a packet arrives, the firewall checks its header (Source IP, Destination IP, Port Number, Protocol) and allows it if it 
matches the predefined rules.

- **_Limitation:_**
It cannot determine whether a packet is a response to a legitimate request or an attempt by an attacker to send malicious traffic.

- **_Attack Vector:_**
Attackers can easily bypass it using packet spoofing or specially crafted packets.

### Stateful Firewall
A stateful firewall is much more intelligent. It tracks the state of each connection passing through the network.

- **_Operation:_**
It maintains a State Table (or session table). When a device inside the network sends a request (e.g., opening a website), the firewall records that information in the table. 
When data returns from the website, the firewall checks the table to verify whether the request was legitimate.

- **_Security Advantage:_**
If a packet arrives without a corresponding record (request) in the session table, the firewall immediately drops it. This makes it much more effective against port scanning and 
DDoS attacks.

## Real-World Attack Scenario
Imagine an attacker performing a TCP Stealth Scan (SYN Scan) to find open ports on your server. 

- **_Stateless Firewall:_** The firewall sees the SYN packet. If your rule says "Allow Port 80," the packet goes through. The attacker learns the port is open and begins their exploit.
- **_Against a Stateful Firewall:_** The firewall sees the SYN packet. It recognizes that this is an attempt to start a new connection. It can apply more complex logic, 
such as Rate Limiting, to see if one IP is hitting 1,000 ports per second. If the behavior is suspicious, it kills the connection and logs the IP as a threat.

### Summary
Think of a Stateless firewall like a security guard who only checks IDs. If you have the right ID, you get in, regardless of why you are there. A Stateful firewall is like a 
security guard with a guest list. Even if you have an ID, if you aren't on the list (the State Table) and nobody inside invited you, you aren't getting past the door.
